# skillpack Design Spec

**Round**: 81
**Date**: 2026-04-14
**Language**: Go 1.21+
**Category**: Agent Skill Packaging / Lifecycle / Lockfile

## Mission
A package manager, lockfile, and bundler for Claude Code / Cursor / AGENT.md agent skills. Resolve, pin, hash, bundle, sign — single static Go binary.

## Why
At least 6 trending repos cluster around the agent-skills wedge with combined stars > 100K, but **zero** lifecycle tooling exists. opencli is a runtime dispatcher (different layer). googleworkspace/cli ships its own bundle (vendor-locked). awesome-claude-code is a hand-curated markdown list. Nothing resolves dependencies, hashes content, or generates lockfiles for skills.

## Common Skill Record (canonical)

```go
type Skill struct {
    Name        string            // unique id within workspace
    Version     string            // semver, normalized (no v-prefix)
    Description string
    Format      Format            // SkillMD | CursorRules | AgentMD | SkillYAML
    SourcePath  string            // origin file (rel to workspace)
    Body        string            // canonicalized body (LF, no trailing ws)
    Frontmatter map[string]string // normalized k/v
    Requires    []Constraint      // semver constraints on other skills
    Tools       []string          // declared tool deps (optional)
    License     string
    Author      string
    Hash        string            // sha256 of canonical bytes (hex)
}
```

## Parser Normalization (deterministic)
1. Read raw file with explicit utf-8.
2. Convert CRLF → LF.
3. Strip BOM if present.
4. Split frontmatter (if any) and body.
5. Parse frontmatter into ordered map; sort keys for canonicalization.
6. Trim body trailing whitespace per line, ensure single trailing LF.
7. Build canonical bytes: `format\n` + `name=...\n` + ... + `---\n` + body.
8. Compute sha256 over canonical bytes.

## Semver Resolver
- Wrap `golang.org/x/mod/semver`.
- Accept versions with or without `v` prefix; normalize internally to `v` prefix for comparison, output without prefix.
- Constraint syntax: `^1.2.3` (caret, ≥1.2.3 <2.0.0), `~1.2.3` (tilde, ≥1.2.3 <1.3.0), `>=1.2.3`, `1.2.3` (exact), `*` (any).
- Resolver: topological sort over `Requires` with conflict detection. Deterministic ordering via lexicographic name tiebreak.
- Exits cleanly on cycles with a `cycle: A → B → A` message.

## Lockfile Format (`skillpack.lock`)

JSON (pretty, sorted keys, LF line endings, trailing newline):

```json
{
  "version": 1,
  "generated_by": "skillpack",
  "skills": [
    {
      "name": "code-review",
      "version": "1.2.0",
      "format": "skill.md",
      "hash": "sha256:abc123...",
      "source": "skills/code-review/SKILL.md",
      "requires": ["base-agent ^1.0.0"]
    }
  ]
}
```

- No timestamps. No machine-specific paths.
- `skills` array sorted by name, then version.
- `requires` sorted lexicographically.
- Marshal with custom encoder that sorts map keys.
- Always ends with `\n`.

## Tarball Bundler (`*.skl`)
- `archive/tar` + `compress/gzip`.
- File order: sorted by name.
- Header normalization:
  - `ModTime`: fixed `1970-01-02 00:00:00 UTC` (epoch+1d to avoid zero-mtime quirks)
  - `Uid/Gid`: 0
  - `Uname/Gname`: ""
  - `Mode`: 0644 for files, 0755 for dirs
  - `Format`: tar.FormatPAX (deterministic across platforms)
- Includes a top-level `manifest.json` with the resolved lockfile snapshot.
- Reject symlinks, devices, hardlinks, absolute paths in entries.
- Path safety: every entry must `filepath.Clean` to a relative path with no `..` segments.

## Signer
- ed25519 detached signature over the tarball bytes.
- `skillpack sign --key key.ed25519 mypack.skl` → writes `mypack.skl.sig`.
- `skillpack verify --pubkey pub.ed25519 mypack.skl` → checks signature.
- Key files are raw 64-byte (private) / 32-byte (public) base64-encoded.

## Verify Mode
- `skillpack verify --lockfile skillpack.lock --root ./skills`
- Re-parses every declared skill, recomputes hash, compares with lockfile.
- Distinct exit codes:
  - `0` PASS
  - `1` DRIFT (hash mismatch / version mismatch)
  - `2` PARSE error (malformed input)
  - `3` IO error (missing file, permission denied)
  - `4` INTERNAL error
- Outputs JSON or human-readable (default human, `--json` flag).

## CLI Commands
| Command | Purpose |
|---|---|
| `skillpack init` | scaffold `skillpack.yaml` workspace manifest |
| `skillpack add <path>` | add a skill source dir to workspace |
| `skillpack resolve` | run resolver, print install order (dry run) |
| `skillpack install` | resolve + write `skillpack.lock` |
| `skillpack verify` | CI mode: hash drift detection |
| `skillpack bundle <out>` | produce `*.skl` tarball |
| `skillpack sign --key X <pkg>` | ed25519 detached signature |
| `skillpack lock` | rewrite lockfile from current state |

## Error Wrapping
- Each package defines typed errors: `parser.Error`, `resolver.Error`, etc.
- `errors.Is` / `errors.As` for classification.
- `cli.Run` translates error type → exit code.

## Determinism Checklist
- [ ] Lockfile JSON: sort keys (custom marshaler), LF, trailing newline.
- [ ] Tarball: sorted file order, fixed mtime, uid/gid 0, PAX format.
- [ ] Hash canonicalization: CRLF→LF, strip BOM, trim trailing whitespace, sort frontmatter keys.
- [ ] No `time.Now()` in any hashable path.
- [ ] No `map` iteration without `sort.Strings(keys)`.
- [ ] Two consecutive runs must produce byte-identical lockfile and tarball.

## Test Plan (~150 tests target)
- parser package: ~40 tests (4 formats × 10 cases each: happy, missing FM, CRLF, empty body, bad YAML, BOM, comments, semver in version, missing name, unicode body)
- semver package: ~25 tests (constraint matching, edge cases, normalization, cycle detection)
- resolver package: ~15 tests (linear chain, diamond, cycle, missing dep, version conflict, deterministic ordering)
- hasher package: ~10 tests (canonical bytes, CRLF normalization, frontmatter ordering, idempotence)
- lockfile package: ~15 tests (round-trip, sort order, deterministic output, malformed input)
- bundle package: ~15 tests (tarball determinism, path safety, manifest inclusion, two-run byte equality)
- signer package: ~10 tests (sign/verify happy, bad key, tampered file, key parsing)
- verify package: ~10 tests (clean, drift, parse error, io error, exit code mapping)
- cli/integration: ~15 tests (end-to-end init→add→install→verify→bundle→sign cycle)

## Dependencies
- `gopkg.in/yaml.v3` — frontmatter parsing
- `golang.org/x/mod/semver` — semver primitives
- `github.com/spf13/cobra` — CLI

That's it. Everything else is std lib.

## Build
```
go build -ldflags="-s -w" -o skillpack ./cmd/skillpack
```
Target: < 15 MB static binary on linux-amd64, similar on windows/darwin.
